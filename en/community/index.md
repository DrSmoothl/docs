### Contributing

You are welcome to contribute to MaiBot's development! Before submitting a contribution, please read the [Contribution Guide](https://github.com/Mai-with-u/MaiBot/blob/main/docs/CONTRIBUTE.md).

Ways to contribute:
- **Submit an Issue**: Report a bug or suggest a feature
- **Submit a Pull Request**: Fix a bug or implement a new feature
- **Improve Documentation**: Supplement and refine project documentation
- **Develop Plugins**: Develop third-party plugins for MaiBot

### Issue Feedback

When submitting an Issue on GitHub, please:
1. Use a clear title to describe the problem
2. Provide reproduction steps and environment information (OS, Python version, MaiBot version)
3. Attach relevant log output
4. Check if an Issue for the same problem already exists

## Design Philosophy

The core design philosophy of MaiBot comes from its founder, SengokuCola (千石可乐):

> This project was originally just to add a few extra features to the Niuniu bot, but the features kept growing, and eventually, it was decided to rewrite it. The goal is to create a "living entity" active in QQ group chats. The purpose is not to build a feature-rich robot, but rather a human-like presence that feels as real as possible.

> The functional design philosophy of the program is based on a core principle: "Most alike, not best."

> If humans truly need an AI to accompany them, not everyone wants a perfect, problem-solving "helpful assistant"; some might prefer an error-prone "life form" with its own perceptions and thoughts.
