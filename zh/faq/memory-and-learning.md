# 记忆与学习

## 怎样开启长期记忆？

在 WebUI 的长期记忆设置中启用 A_Memorix，并配置可用的 embedding 模型。配置文件中的总开关是 `[a_memorix.plugin]` 下的 `enabled`。

详细配置见[A_Memorix 配置](../manual/configuration/amemorix-config.md)。

## 为什么提示“关系向量未启用”？

普通记忆向量和关系向量是不同能力。关系向量当前可以使用，但默认关闭；看到这条提示不表示整个长期记忆系统失效。

关系向量由 `a_memorix.retrieval.relation_vectorization` 下的启用、回填和导入写入设置控制。开启前应确保 embedding 服务稳定，并了解历史数据回填会产生额外请求。

## 关系向量关闭后，为什么仍然有 embedding 请求？

embedding 还用于段落、实体和长期记忆检索。关闭关系向量只是不为关系数据写入或查询向量，不会关闭整个记忆向量系统。

## embedding 模型可以不配置吗？

如果启用了长期记忆、知识导入或依赖向量检索的功能，就需要配置 embedding 模型。完全不使用这些功能时可以不配置，但相关能力将不可用或在调用时报告错误。

## 记忆为什么会把人物或主客体记反？

先在记忆管理中查看来源消息，确认原始表达是否存在歧义，再检查抽取或写回时使用的模型日志。如果来源明确而抽取结果错误，可以更换记忆任务使用的模型，并通过反馈纠错或删除后重新写入。

不要仅依据单次错误认定某个型号始终不可用；模型版本和服务商实现都可能变化。

## 黑话需要出现多少次才会学习？

学习不是达到固定次数后必然成功。样本数量、上下文是否一致、表达是否稳定以及学习模型都会影响结果。出现次数太少时通常无法可靠推测含义；随着一致样本增加，结果才可能逐渐稳定。

学习结果不正确时，可以在 WebUI 中检查和修改，不建议在文档中承诺固定次数或百分之百准确。

## 怎样让多个群共享表达或黑话？

使用表达或黑话互通组，把需要共享的聊天流放入同一组。`platform` 和 `item_id` 支持的通配规则及完整字段见[Bot 配置](../manual/configuration/bot-config.md)。使用全局通配前应确认确实希望所有群共享学习结果。
