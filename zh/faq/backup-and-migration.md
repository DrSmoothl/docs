# 数据与迁移

## 迁移或重装前需要备份什么？

至少备份以下内容：

**`data/`** — 数据库、记忆、表情包及运行数据。

**`config/`** — Bot、模型和其他主配置。

**`plugins/`** — 已安装插件及插件目录内的配置和数据。

如果适配器或一键包在 MaiBot 目录外保存配置，也要单独备份。不要把 `plugins` 误认为位于 `config` 目录。

## 怎样迁移到另一台设备或另一个目录？

1. 停止 MaiBot、适配器和相关启动器。
2. 备份完整目录，并记录当前版本。
3. 在目标位置准备兼容版本和运行环境。
4. 恢复 `data`、`config`、`plugins` 及外部适配器配置。
5. 检查配置中的绝对路径、端口、权限和网络地址。
6. 启动后查看配置升级、数据库迁移和插件加载日志。

确认新位置运行正常后再删除旧副本。

## 恢复备份后无法启动怎么办？

重点检查源版本与目标版本是否兼容、配置是否需要升级、数据库迁移是否失败、插件是否支持当前版本，以及文件权限和绝对路径是否仍然有效。

不要在没有额外备份的情况下删除数据库或覆盖原始备份。

## 本地模型提示禁止访问非公网地址怎么办？

这是 WebUI 的出站 URL 安全校验。确认目标确实是可信的本地或内网模型服务后，可以关闭 `enforce_public_outbound_url`。关闭后 WebUI 将允许访问内网地址，也会降低防止服务端请求伪造的保护强度。

如果模型服务运行在 Docker 或另一台设备上，还要确认地址不是错误地使用了当前容器的 `127.0.0.1`。
