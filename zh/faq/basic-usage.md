# 基础使用

## MaiBot 需要付费吗？

MaiBot 本身是开源软件，不收取软件使用费。实际运行通常需要调用大语言模型、视觉模型或 embedding 服务，是否收费及价格由对应服务商决定。

使用量主要受聊天频率、上下文长度、启用的功能和模型价格影响。可以通过降低发言频率、选择适合的模型和关闭暂时不用的功能控制开销。

## 应该选择什么模型？

没有适合所有人的固定答案。选择时重点确认：

- Planner 是否具备可靠的推理和工具调用能力。
- Replyer 的语言质量是否符合预期。
- 视觉任务使用的模型是否支持图片输入。
- 长期记忆使用的是否为 embedding 模型。
- API 服务的价格、延迟、稳定性和地区可用性。

具体角色和配置方式见[模型配置](../manual/configuration/model-config.md)。不建议只依据一份容易过时的型号推荐表选择模型。

## 一个 MaiBot 可以服务多个群聊吗？

可以。具体能接入哪些群聊和私聊由适配器的聊天过滤配置决定。使用白名单时，只有名单中的群聊或用户会被处理；使用黑名单时，名单中的对象会被忽略。

## MaiBot 支持哪些平台？

平台支持由已安装的适配器决定。当前文档包含 NapCat、SnowLuma、GoCQ、Telegram 和 Discord 等适配器，实际可用能力以对应适配器版本为准。

## 第一次使用需要注意什么？

建议先完成最小配置并验证消息链路，再逐步开启长期记忆、插件和其他高级功能。遇到异常时优先查看日志，同时记录 MaiBot、适配器和一键包版本，避免一次修改多个无关配置。

## 提问时需要提供什么？

请至少提供操作系统、MaiBot 版本、部署方式、适配器版本、完整错误日志、相关配置和复现步骤。公开日志或配置前应删除 API Key、访问令牌、Cookie 等敏感信息。
